## Tier 2 — Scripted browser checks (Playwright)

All items are implemented as `test.step()` or `test()` blocks in
`tests/e2e/a11y.spec.js` and run as part of `npm run test:a11y`.

| # | Check | Spec location | Status |
|---|-------|---------------|--------|
| 3 | Keyboard-only authoring flow | `test.step('citation input and add button work from the keyboard')` + `test.step('citation row and action controls are keyboard reachable')` | ✅ implemented |
| 4 | Focus management after Add | `test.step('focus moves into block after keyboard-activated Add')` — asserts `document.activeElement` is not `body`/`html` | ✅ implemented |
| 5 | Modal / popover focus trap and restore | `test.step('focus restores to chevron after citation form collapse')` — asserts `document.activeElement === chevronButton` | ✅ implemented |
| 6 | Visible focus indicators | `test.step('interactive controls expose a visible focus indicator')` — checks computed `outline` and `box-shadow` via JS | ✅ implemented |
| 7 | Forced-colors (Windows High Contrast) | `test('block passes axe scan under forced-colors media')` — `page.emulateMedia({ forcedColors: 'active' })` + axe scan | ✅ implemented |
| 8 | 400% zoom / reflow at 320 px | `test('published bibliography reflows at 320 px width without horizontal scroll')` — asserts `scrollWidth <= innerWidth` | ✅ implemented |
| 9 | BAC error indicator fires on empty block | `test('BAC error indicator appears on empty block, clears when complete')` — checks `.ba11y-block-indicator--error`; skips gracefully if BAC absent | ✅ implemented |
| 10 | BAC checks pass on complete block | Same test — adds citation + heading text, asserts both error and warning indicators are gone | ✅ implemented |

### Notes on Tier 2 coverage

- Items 3–6 are `test.step()` calls nested inside the single serial
  `test('keyboard behavior and axe scans pass in editor and frontend')`.
- Items 7, 8, and 9–10 are independent top-level `test()` blocks.
- The BAC test (items 9–10) uses `test.skip()` when
  `window.BlockAccessibilityChecks` is absent, so it passes cleanly in
  Playground environments where the BAC plugin is not installed.
- No Tier 2 gaps were found. No spec changes were required as part of
  this review.
