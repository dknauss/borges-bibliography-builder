# i18n Process

## POT regeneration

After modifying PHP or JS strings, regenerate the template with WP-CLI:

```bash
wp i18n make-pot . languages/borges-bibliography-builder.pot \
  --domain=borges-bibliography-builder \
  --exclude=node_modules,vendor,packages \
  --headers='{"Project-Id-Version":"Borges Bibliography Builder","Report-Msgid-Bugs-To":"https://github.com/dknauss/borges-bibliography-builder/issues"}'
```

The POT is checked into `languages/`. Commit it alongside any source string changes so translators and translate.wordpress.org stay in sync.

## MO compilation

After editing a PO file, compile the binary MO:

```bash
msgfmt languages/borges-bibliography-builder-fr_FR.po \
       -o languages/borges-bibliography-builder-fr_FR.mo
```

Or compile all at once:

```bash
for f in languages/*.po; do
  msgfmt "$f" -o "${f%.po}.mo"
done
```

## JS JSON artifacts

WordPress loads block-editor JS translations from JSON files generated by WP-CLI. When translations are bundled locally (not via WordPress.org language packs), run:

```bash
wp i18n make-json languages/borges-bibliography-builder-fr_FR.po \
  --no-purge \
  --pretty-print
```

This produces one or more files like `languages/borges-bibliography-builder-fr_FR-<hash>.json` that WordPress discovers automatically via `wp_set_script_translations()`. The block registers its script handle in `build/index.js`; the `textdomain` field in `block.json` is `borges-bibliography-builder`.

No JS JSON files are currently bundled because the plugin relies on WordPress.org language packs for JS strings in deployed locales. If bundled translations are added back for a locale, run `make-json` and commit the resulting files.

## Bundled seed files vs official language packs

The `languages/` directory contains seed PO/MO files for 19 locales. These are import material for translators and local development only. They are not official WordPress.org language packs and must not be advertised as such.

Official language packs are generated by WordPress.org after the **Stable** translation project for a locale reaches the required approval threshold on [translate.wordpress.org](https://translate.wordpress.org/projects/wp-plugins/borges-bibliography-builder/). The canonical list of published packs is the **Languages** tab on the plugin's WordPress.org page.

## translate.wordpress.org approval path

1. Strings are submitted to translate.wordpress.org automatically when a new plugin version is tagged (WP.org SVN deploy triggers string import).
2. Community translators or locale PTEs (Plugin Translation Editors) submit translations for the **Stable** sub-project.
3. A locale GTE (General Translation Editor) or the plugin author (if granted PTE status) approves strings until the locale reaches 100 % of the Stable string set.
4. WordPress.org generates a language pack; it becomes available via `wp language plugin install` and automatic updates.

To accelerate a specific locale, contact the locale team via their GlotPress discussion or the #polyglots channel on WordPress.org Slack, referencing this plugin's slug `borges-bibliography-builder`. Professionally reviewed translations can be submitted through the CLPTE (Community Language Pack Translation Editor) workflow — do not submit unreviewed machine translations.
